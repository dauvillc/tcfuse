# TC-Fuse model architecture

Read this before touching `src/tcfuse/models/`, adding or benchmarking a backbone, or changing the embedding/encoder/decoder interface.

## Architecture philosophy

The framework is **architecture-agnostic at the backbone level**. The embedding layer (value + coordinate ŌåÆ token) and the task heads (decoder) are fixed interfaces; the encoder between them is swappable.

```
[Source 1: values + coords] ŌöĆŌöĆŌöÉ
[Source 2: values + coords] ŌöĆŌöĆŌö╝ŌöĆŌöĆŌ¢║ [Source Embeddings] ŌöĆŌöĆŌ¢║ [Encoder (swappable)] ŌöĆŌöĆŌ¢║ [Task Head]
[Source N: values + coords] ŌöĆŌöĆŌöś
```

**The embedding layer (implemented):** `tcfuse.models.encoders` patch-embeds each `TorchSource` into an `EmbeddedSource` carrying a single `features` tensor ŌĆö `(B, D)` SCALAR, `(B, El, D)` PROFILE, `(B, Eh, Ew, D)` FIELD, with `El/Eh/Ew = L/H/W // patch_size`. `MultiSourceEncoder` (Hydra `_partial_: true`, allocates one per-source encoder from `sources_metadata`) maps a `WindowBatch` ŌåÆ `EmbeddedBatch` (`sources` + `is_target` only; mask dropped). Each encoder also **adds a spatio-temporal Fourier positional encoding** to its value tokens (`tcfuse.models.encoders.positional.SpatioTemporalEncoding`): per-axis deterministic, log-spaced sinusoidal Fourier features over lat/lon (degrees), altitude/depth for PROFILE (meters), and time (`[day/366, minute/1440]`), concatenated and linearly projected to `D`, then summed onto the value embedding. For PROFILE/FIELD, per-pixel/per-level coords are average-pooled to per-patch centers (replicate-padded) before encoding. Knobs (frequency count, per-group wavelength ranges, `enabled`) live in `CoordEncodingConfig`, threaded through `MultiSourceEncoder` and both backbones; defaults applied when unset. NaN-fill in values/coords/time (absent or masked slots) is zeroed **inside the encoder** (`torch.nan_to_num` in each `SourceEncoder.forward` for values, in `SpatioTemporalEncoding.forward` for coords/time), so each backbone owns its NaN handling and there is no separate pre-backbone cleaning step. A token-validity mask is still a deferred follow-up.

**The MoTiF embedding layer (implemented):** `tcfuse.models.encoders_motif` is a parallel embedding package for the planned MoTiF backbone (a diffusion-transformer-style architecture where coordinate embeddings are injected as positional conditioning at **every** layer, instead of being added once to the value tokens). Each source is embedded into **two tensors** sharing the same token layout but independent embedding dims: a value-only tensor (`(B, Dv)` SCALAR, `(B, El, Dv)` PROFILE, `(B, Eh, Ew, Dv)` FIELD ŌĆö same patchify as `encoders`, no positional term added) and a standalone coordinate tensor (`(ŌĆ”, Dc)`, same spatial dims; `MotifEmbeddedSource` validates the shared-spatial-dims contract). The coordinate tensor comes from `CoordEmbedding` (`encoders_motif.positional`): the same per-axis log-spaced sin/cos Fourier features over lat/lon (+ altitude for PROFILE) and time, projected by a learnable `nn.Linear` to `Dc`; per-pixel/per-level coords are average-pooled to patch centers first. Knobs live in `MotifCoordEncodingConfig` (no `enabled` switch ŌĆö the separate coord tensor is the architecture's premise). `MotifMultiSourceEncoder` mirrors `MultiSourceEncoder`'s dispatch (`WindowBatch` ŌåÆ `MotifEmbeddedBatch`). Deliberate code redundancy with `encoders/`; nothing existing was modified. The MoTiF backbone and its decoder counterpart are not yet implemented.

**The un-embedding layer (implemented):** `tcfuse.models.decoders` is the symmetric inverse, mapping an `EmbeddedBatch` ŌåÆ `DecodedBatch` (`DecodedSource.values` matching the original `TorchSource.values` layout; `is_target` passed through). `ScalarDecoder`/`ProfileDecoder` are exact shape inverses (`nn.Linear` / `nn.ConvTranspose1d`) of their encoders; `FieldDecoder` instead uses sub-pixel convolution (`Conv2d` + `PixelShuffle`) with ICNR-initialized weights to avoid checkerboard artifacts on image-like FIELD data. `MultiSourceDecoder` mirrors `MultiSourceEncoder`'s Hydra-partial/per-source-allocation pattern. Not yet wired into `BaseLightningModule` or any task head's cross-source querying mechanism.

## The encoder interface

- Input: a list of token sequences, one per source, each of shape `(B, N_i, D)` where `N_i` is the number of tokens for source `i` and `D` is the embedding dimension.
- Output: a representation that the task head can query ŌĆö exact form depends on architecture (latent array for Perceiver, CLS token for ViT-style, etc.).
- The encoder must be instantiable from a Hydra config node, using `_partial_: true` so that `BaseLightningModule` can pass `sources_metadata` to the constructor at runtime (allowing backbones to allocate per-source parameters from channel counts). The model config lives in `conf/model/` and is imported into the lightning module config via a defaults package-override entry: `- /model@model: <name>`.

**The plain single-sequence transformer backbone (implemented):** `tcfuse.models.transformer.backbone.SingleSequenceTransformerBackbone` is the first concrete backbone candidate. It owns its own `MultiSourceEncoder` / `MultiSourceDecoder` internally (built from `sources_metadata` + `embed_dim` + `patch_size` passed to its constructor), so externally it satisfies `WindowBatch -> WindowBatch`. Internally: every source's embedded tokens are flattened to `(B, Ls, D)` and concatenated into one `(B, L_total, D)` multi-source sequence; a stack of pre-norm `TransformerBlock`s (`tcfuse.models.transformer.block`) processes the whole sequence with dense self-attention (`tcfuse.models.transformer.attention.MultiHeadSelfAttention`, built on `torch.nn.functional.scaled_dot_product_attention`, no mask ŌĆö see token-validity caveat below) and a position-wise feed-forward sub-layer (`tcfuse.models.transformer.feedforward.FeedForward`); the sequence is then split and reshaped back per source and decoded. Config: `conf/model/transformer.yaml`.

**The Perceiver IO backbone (implemented):** `tcfuse.models.perceiver.backbone.PerceiverIOBackbone` is the second concrete backbone candidate, fully independent of the transformer modules (deliberate code redundancy). Like the transformer it owns its own `MultiSourceEncoder` / `MultiSourceDecoder` and satisfies `WindowBatch -> WindowBatch`. Internally: source tokens are flattened/concatenated into one `(B, L, D)` sequence `X`; a learned latent array `Z` of shape `(M, Dz)` (`nn.Parameter`, expanded to `(B, M, Dz)`) is the working memory. An encode cross-attention (`CrossAttentionBlock`, queries = `Z`, keys/values = `X`) writes the sources into the latents; a stack of pre-norm `LatentBlock`s (self-attention + MLP on `Z`, dims `Dz`) does the heavy compute on the short latent sequence; a decode cross-attention (`CrossAttentionBlock`, queries = `X`, keys/values = `Z`) reads the latents back out into `X'`, which is split/reshaped per source and decoded. All attention uses `torch.nn.functional.scaled_dot_product_attention` (`tcfuse.models.perceiver.attention.SelfAttention` / `CrossAttention`); cross-attention blocks are Perceiver-IO style (pre-LN on query and kv, residual on the query stream, then a pre-norm MLP). Because the decode queries come from `X` and masked sources were zeroed before embedding, those queries carry only coordinate information ŌĆö reconstructing them from `Z` leaks no value information. `cross_num_heads` must divide both `embed_dim` and `latent_dim`; `num_heads` must divide `latent_dim`. Config: `conf/model/perceiver.yaml`.

**The two-sequence cross-attention transformer backbone (implemented):** `tcfuse.models.cross_transformer.backbone.CrossSequenceTransformerBackbone` is the third concrete backbone candidate, built to avoid the Perceiver's latent compression (which loses too much detail for high-resolution target reconstruction). Like the others it owns its own `MultiSourceEncoder` / `MultiSourceDecoder` and satisfies `WindowBatch -> WindowBatch`, and is fully self-contained (its own `attention`/`block`/`feedforward` modules ŌĆö deliberate redundancy). Internally: source tokens are flattened/concatenated into one `(B, L, D)` sequence `X`; a per-token target flag (built from `EmbeddedBatch.is_target`, which is per-sample `(B,)` per source) splits each sample into a **target sequence** (its single target source's tokens, as queries) and an **input sequence** (all its non-target sources, as keys/values). Because the target source ŌĆö hence each sample's target/input token counts ŌĆö differs within a batch, the ragged per-sample streams are gathered (stable argsort on the flag) front-aligned into padded `(B, L_max, D)` tensors with validity masks; the padding is masked out of attention. A stack of `CrossSeqBlock`s updates only the target stream: pre-LN cross-attention (target queries the fixed input keys/values), pre-LN self-attention within the target, then a pre-LN MLP ŌĆö `(cross, self, mlp)` repeated `num_layers` times. The processed target tokens are scattered back to their original positions (`order` is a per-row permutation, so indices never collide; padding columns write the original value as a no-op, also keeping padding-row garbage out of the output), then split/reshaped per source and decoded. Masked target queries carry only coordinate information (values zeroed before embedding), so reconstructing them from the inputs leaks no values. Attention masks only the structural ragged-padding ŌĆö genuinely-absent (non-target) input sources stay in the input sequence (zeroed), consistent with the token-validity caveat below. Only constraint: `num_heads` must divide `embed_dim`. Config: `conf/model/cross_transformer.yaml`; runnable experiment: `conf/experiment/pmw_gmi_cross_transformer.yaml`.

## Candidate architectures to benchmark

Extend as needed:

- Plain single-sequence transformer (implemented, see above) ŌĆö baseline to benchmark others against.
- Perceiver / Perceiver IO (implemented, see above).
- Cross-attention Transformer (two-sequence inputŌåÆtarget; implemented, see above).
- Hierarchical windowed attention (Swin-style, per source + cross-source).

## Self-supervised pre-training task

Randomly mask one source at training time; reconstruct its values from all remaining sources, using only its coordinates and instrument metadata as queries. This is the default pre-training objective. Supervised fine-tuning follows for specific tasks.
